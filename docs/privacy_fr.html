📋 Finalité de l'application

Driverslog est utilisé pour l'enregistrement et la gestion privés ou professionnels des déplacements en véhicule. L'application permet de consigner les adresses de départ et de destination, les distances et les données du conducteur, et peut exporter des rapports de trajets en PDF ou CSV.

L'application est conçue pour un usage personnel. Aucun compte utilisateur n'est créé et aucune donnée n'est stockée dans le cloud.

💾 Données stockées localement

Toutes les données saisies sont stockées exclusivement en local sur l'appareil (base de données SQLite). Cela comprend :

  • Données de trajet (date, départ, destination, kilomètres, conducteur)
  • Carnet d'adresses (rue, ville, coordonnées)
  • Données du conducteur (nom)
  • Paramètres de l'application

Ces données ne quittent pas automatiquement l'appareil et ne sont pas transmises au développeur.

✅ Pas de stockage cloud

🌐 Services externes (accès réseau)

L'application utilise les services externes suivants, exclusivement sur initiative de l'utilisateur (par ex. lors du calcul d'un itinéraire) :

Nominatim – OpenStreetMap

Pour convertir des adresses en coordonnées (géocodage), les textes d'adresse sont transmis au service public Nominatim de la Fondation OpenStreetMap. Aucune donnée personnelle n'est transmise, uniquement le texte d'adresse saisi.
Confidentialité : osmfoundation.org/wiki/Privacy_Policy

OSRM – Open Source Routing Machine

Pour calculer l'itinéraire réel, les coordonnées de départ et de destination sont transmises au serveur de démonstration OSRM (router.project-osrm.org). OSRM est gratuit et ne nécessite ni compte ni clé API. Seules les coordonnées GPS sont transmises – aucune donnée personnelle.
Projet : project-osrm.org

OpenRouteService (ORS) – optionnel

Alternativement, une clé API personnelle d'OpenRouteService peut être enregistrée dans les paramètres. Si une clé est saisie, les coordonnées de départ et de destination sont transmises à OpenRouteService (HeiGIT, Heidelberg) au lieu d'OSRM. Le service offre 500 requêtes/jour gratuitement. Sans clé, OSRM est utilisé.
Confidentialité : openrouteservice.org/privacy-policy

Ces trois services ne sont appelés que lorsque l'utilisateur déclenche activement un calcul de distance. Aucune requête automatique en arrière-plan n'est effectuée. OSRM est utilisé par défaut ; ORS uniquement si une clé API est saisie.

📶 Synchronisation Wi-Fi

L'application propose une synchronisation optionnelle entre deux appareils sur le même réseau Wi-Fi (par ex. smartphone Android et PC Windows). Les données de trajet sont échangées directement entre les deux appareils – sans passer par Internet ni par des serveurs externes.

La transmission s'effectue en clair (TCP) sur le réseau local. Aucune donnée n'est partagée avec des tiers.

🔒 Permissions Android

  • INTERNET – pour le géocodage Nominatim, le routage OSRM et le routage ORS optionnel
  • ACCESS_WIFI_STATE / ACCESS_NETWORK_STATE – pour la détection de synchronisation Wi-Fi
  • CHANGE_WIFI_MULTICAST_STATE – pour la découverte d'appareils via diffusion UDP sur le réseau local
  • READ_EXTERNAL_STORAGE / WRITE_EXTERNAL_STORAGE (Android ≤12) – accès au fichier de base de données
  • NEARBY_WIFI_DEVICES (Android 13+) – pour la synchronisation Wi-Fi directe

Aucune permission pour l'appareil photo, le microphone, la localisation, les contacts ou d'autres données sensibles n'est demandée.

🚫 Pas de partage de données

Il n'y a aucun partage de données personnelles avec le développeur, les annonceurs ou d'autres tiers. L'application ne contient aucun SDK d'analyse, aucune publicité et aucun suivi.

✅ Pas de suivi  ·  ✅ Pas de publicité  ·  ✅ Pas d'analyse

⚖️ Vos droits

Toutes les données étant stockées localement sur votre appareil, vous en avez le contrôle total à tout moment. Vous pouvez supprimer toutes les données directement dans l'application ou supprimer complètement les données de l'application via les paramètres Android.

Pour toute question concernant la protection des données, contactez-nous par e-mail à : info.kloesel@gmx.de

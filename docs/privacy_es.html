📋 Finalidad de la aplicación

Driverslog se utiliza para el registro y gestión privado o profesional de viajes en vehículo. La aplicación permite registrar direcciones de origen y destino, distancias y datos del conductor, y puede exportar informes de viajes en PDF o CSV.

La aplicación está diseñada para uso personal. No se crean cuentas de usuario y no se almacenan datos en la nube.

💾 Datos almacenados localmente

Todos los datos introducidos se almacenan exclusivamente de forma local en el dispositivo (base de datos SQLite). Esto incluye:

  • Datos de viaje (fecha, origen, destino, kilómetros, conductor)
  • Libreta de direcciones (calle, ciudad, coordenadas)
  • Datos del conductor (nombre)
  • Configuración de la aplicación

Estos datos no salen del dispositivo automáticamente y no se transmiten al desarrollador.

✅ Sin almacenamiento en la nube

🌐 Servicios externos (acceso a la red)

La aplicación utiliza los siguientes servicios externos, exclusivamente por iniciativa del usuario (p. ej. al calcular una ruta):

Nominatim – OpenStreetMap

Para convertir direcciones en coordenadas (geocodificación), los textos de dirección se transmiten al servicio público Nominatim de la Fundación OpenStreetMap. No se transmiten datos personales, solo el texto de dirección introducido.
Privacidad: osmfoundation.org/wiki/Privacy_Policy

OSRM – Open Source Routing Machine

Para calcular la ruta de conducción real, las coordenadas de origen y destino se transmiten al servidor de demostración público de OSRM (router.project-osrm.org). OSRM es gratuito y no requiere cuenta ni clave API. Solo se transmiten coordenadas GPS, sin datos personales.
Proyecto: project-osrm.org

OpenRouteService (ORS) – opcional

Alternativamente, se puede guardar una clave API personal de OpenRouteService en la configuración. Si se introduce una clave, las coordenadas de origen y destino se transmiten a OpenRouteService (HeiGIT, Heidelberg) en lugar de a OSRM. El servicio ofrece 500 solicitudes/día de forma gratuita. Sin clave, se utiliza OSRM.
Privacidad: openrouteservice.org/privacy-policy

Los tres servicios solo se invocan cuando el usuario activa activamente un cálculo de distancia. No se producen solicitudes automáticas en segundo plano. OSRM se utiliza de forma predeterminada; ORS solo si se introduce una clave API.

📶 Sincronización Wi-Fi

La aplicación ofrece sincronización opcional entre dos dispositivos en la misma red Wi-Fi (p. ej. smartphone Android y PC Windows). Los datos de viaje se intercambian directamente entre los dos dispositivos, sin pasar por Internet ni por servidores externos.

La transmisión se realiza sin cifrar (TCP) en la red local. No se comparten datos con terceros.

🔒 Permisos de Android

  • INTERNET – para geocodificación Nominatim, enrutamiento OSRM y enrutamiento ORS opcional
  • ACCESS_WIFI_STATE / ACCESS_NETWORK_STATE – para detección de sincronización Wi-Fi
  • CHANGE_WIFI_MULTICAST_STATE – para descubrimiento de dispositivos mediante difusión UDP
  • READ_EXTERNAL_STORAGE / WRITE_EXTERNAL_STORAGE (Android ≤12) – acceso al archivo de base de datos
  • NEARBY_WIFI_DEVICES (Android 13+) – para sincronización directa Wi-Fi

No se solicitan permisos para cámara, micrófono, ubicación, contactos u otros datos confidenciales del dispositivo.

🚫 Sin compartir datos

No existe ningún intercambio de datos personales con el desarrollador, anunciantes u otros terceros. La aplicación no contiene SDK de análisis, publicidad ni seguimiento.

✅ Sin seguimiento  ·  ✅ Sin publicidad  ·  ✅ Sin análisis

⚖️ Sus derechos

Dado que todos los datos se almacenan localmente en su dispositivo, tiene control total en todo momento. Puede eliminar todos los datos directamente en la aplicación o eliminar completamente los datos de la aplicación a través de la configuración de Android.

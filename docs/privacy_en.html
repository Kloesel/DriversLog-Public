📋 Purpose of the App

Driverslog is used for private or business recording and management of vehicle trips. The app allows logging of start and destination addresses, distances, and driver data, and can export trip reports as PDF or CSV.

The app is designed for personal use. No user accounts are created and no data is stored in the cloud.

💾 Locally Stored Data

All entered data is stored exclusively locally on the device (SQLite database). This includes:

  • Trip data (date, start, destination, kilometers, driver)
  • Address book (street, city, coordinates)
  • Driver data (name)
  • App settings

This data does not leave the device automatically and is not transmitted to the developer.

✅ No cloud storage

🌐 External Services (Network Access)

The app uses the following external services, exclusively on user initiative (e.g. when calculating a route):

Nominatim – OpenStreetMap

To convert addresses to coordinates (geocoding), address texts are transmitted to the public Nominatim service of the OpenStreetMap Foundation. No personal data is transmitted, only the entered address text.
Privacy: osmfoundation.org/wiki/Privacy_Policy

OSRM – Open Source Routing Machine

To calculate the actual driving route, start and destination coordinates are transmitted to the public OSRM demo server (router.project-osrm.org). OSRM is free and requires no account or API key. Only GPS coordinates are transmitted – no personal data.
Project: project-osrm.org

OpenRouteService (ORS) – optional

Alternatively, a personal API key from OpenRouteService can be stored in the settings. If a key is entered, start and destination coordinates are transmitted to OpenRouteService (HeiGIT, Heidelberg) instead of OSRM. The service offers 500 requests/day for free. Without a key, OSRM is used.
Privacy: openrouteservice.org/privacy-policy

All three services are only called when the user actively triggers a distance calculation. No automatic background requests occur. OSRM is used by default; ORS only if an API key is entered.

📶 Wi-Fi Synchronisation

The app offers optional synchronisation between two devices on the same Wi-Fi network (e.g. Android smartphone and Windows PC). Trip data is exchanged directly between the two devices – without going through the internet or external servers.

The transmission takes place unencrypted (TCP) on the local network. No data is shared with third parties.

🔒 Android Permissions

  • INTERNET – for Nominatim geocoding, OSRM routing and optional ORS routing
  • ACCESS_WIFI_STATE / ACCESS_NETWORK_STATE – for Wi-Fi sync detection and Wi-Fi check before routing requests
  • CHANGE_WIFI_MULTICAST_STATE – for device discovery via UDP broadcast on the local network
  • READ_EXTERNAL_STORAGE / WRITE_EXTERNAL_STORAGE (Android ≤12) – access to the database file in external storage
  • NEARBY_WIFI_DEVICES (Android 13+) – for Wi-Fi direct sync without location permission

No permissions for camera, microphone, location, contacts or other sensitive device data are requested.

🚫 No Data Sharing

There is no sharing of personal data with the developer, advertisers or other third parties. The app contains no analytics SDKs, no advertising and no tracking.

✅ No tracking  ·  ✅ No ads  ·  ✅ No analytics
